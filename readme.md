
## ревью
 * по поводу интерфейса: думаю идеологически правильней будет иметь инетрфейс с единственным методом Log, который будет формировать строку из нужных полей и отдавать наружу, нежели для каждой структуры описывать все методы, даже если они не используются
 * посоветую не игнорировать возвращаемую ошибку fmt.Fprintln. Всегда лучше хоть как-то на нее среагировать, чем вообще не узнать о ней. В Go очень любят обрабатывать ошибки (или не очень, но приходится...) :)
 * вместо Fprintln лучше использовать форматированный вывод Fprintf, он нагляднее, чем обычное склеивание строк
